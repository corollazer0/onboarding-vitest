# PRD: TaskFlow - Vue3 + Vitest 온보딩 학습 프로젝트

## 문서 정보

| 항목 | 내용 |
|------|------|
| 문서 버전 | 1.0.0 |
| 작성일 | 2025-01 |
| 대상 | 프론트엔드 팀 (5명) |
| 목적 | Vue3 온보딩 + Vitest TDD 환경 구축 학습 |

---

## 1. 프로젝트 개요

### 1.1 배경 및 목적

**배경**
- 팀원 5명 중 Vue 경험자 2명, 미경험자 3명
- 상반기 5개월 내 Vitest 기반 TDD 환경 구축 필요
- 실무 프로젝트 적용 전 학습용 프로젝트 필요

**목적**
- Vue3, Vite, Vitest, Pinia, Vue Router, Axios의 핵심 기능 학습
- TDD(Test-Driven Development) 개발 문화 내재화
- 커밋 히스토리를 통한 점진적 학습 경험 제공

### 1.2 프로젝트 선정 이유: TaskFlow (할 일 관리 앱)

| 선정 기준 | TaskFlow 적합성 |
|-----------|-----------------|
| 복잡도 | 적절함 - CRUD 기본 + 상태관리 + 라우팅 포함 |
| 학습 범위 | 모든 필수 기술 스택 커버 가능 |
| 확장성 | 단계별 기능 추가로 점진적 학습 가능 |
| 실무 연관성 | 실제 업무 도구와 유사한 패턴 |
| 테스트 용이성 | 명확한 입출력으로 테스트 케이스 작성 용이 |

## 2. 제품 요구사항

### 2.1 사용자 스토리

```
사용자로서, 나는 할 일을 관리하여 업무 효율을 높이고 싶다.
```

### 2.2 핵심 기능 목록

#### Phase 1: 기초 (Week 1-2)

| ID | 기능 | 설명 | 학습 포인트 |
|----|------|------|-------------|

| F-002 | 환경변수 설정 | .env 파일 구성 | import.meta.env 활용 |
| F-003 | 할 일 목록 표시 | 할 일 리스트 렌더링 | ref, reactive, v-for |
| F-004 | 할 일 추가 | 새 할 일 입력 및 추가 | v-model, 이벤트 핸들링 |
| F-005 | 할 일 삭제 | 할 일 항목 삭제 | 배열 조작, emit |

#### Phase 2: 상태관리 (Week 3-4)

| ID | 기능 | 설명 | 학습 포인트 |
|----|------|------|-------------|
| F-006 | Pinia 스토어 구축 | 중앙 상태 관리 | defineStore, state, getters, actions |
| F-007 | 할 일 완료 토글 | 완료 상태 변경 | Pinia actions, 상태 변이 |
| F-008 | 필터링 기능 | 전체/완료/미완료 필터 | computed, getters |
| F-009 | 로컬 스토리지 연동 | 데이터 영속화 | Pinia plugins, watch |

#### Phase 3: 라우팅 (Week 5-6)

| ID | 기능 | 설명 | 학습 포인트 |

| ID | 기능 | 설명 | 학습 포인트 |
